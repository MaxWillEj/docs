## How It Works

1. **Upload API Specs** - Upload/paste/URL or AI-process documents (PDF, DOCX, etc.)
2. **Validate** - Automatic OpenAPI 3.0+ validation with auth detection  
3. **Configure** - Set API credentials and MCP authentication method
4. **Deploy** - Generate MCP server registered in `mcp_server_registry`
5. **Connect** - Use OAuth or JWT to connect AI agents

## MCP Server Authentication

All deployed MCP servers support OAuth 2.1 with PKCE or JWT Bearer authentication.

### OAuth 2.1 Setup (Recommended)

**IMPORTANT:** Remote MCP servers with OAuth must be configured via **Settings > Connectors** in Claude Desktop or Claude.ai, not configuration files.

#### For Claude Desktop:

1. Open Claude Desktop â†’ **Settings â†’ Connectors**
2. Click **"Add custom connector"**
3. Enter the following:
   - **Name**: `Tydli - [your-deployment-slug]`
   - **MCP Server URL**: `https://nqfciqtsrcjorlqcglmq.supabase.co/functions/v1/mcp-router/[your-slug]`
   - **OAuth Client ID**: (from your deployment dashboard)
   - **Authorization URL**: `https://nqfciqtsrcjorlqcglmq.supabase.co/functions/v1/mcp-oauth-server/authorize`
   - **Token URL**: `https://nqfciqtsrcjorlqcglmq.supabase.co/functions/v1/mcp-oauth-server/token`
4. Click **Connect** - your browser will open for authorization

#### For Claude.ai (Web):

1. Go to https://claude.ai/settings/connectors
2. Click **"Add custom connector"**
3. Enter the same information as above
4. Click **Connect**

**Note:** This uses OAuth 2.1 with PKCE (Proof Key for Code Exchange). No client secret is required for public clients.

### JWT Bearer Token (Alternative)

For direct API access without OAuth flow, use JWT bearer tokens configured via your deployment dashboard. The MCP server will validate the JWT on each request.
All deployed MCP servers support OAuth 2.1 and JWT Bearer Token authentication.
